很多人一说做博客，第一反应是买服务器、装面板、配数据库。

这条路当然能走，但它会把注意力从“写什么、怎么持续写”拉到“服务器为什么又要更新、登录口要不要防爆破、备份有没有做”。

如果只是个人博客，我更建议先用静态博客。

## 静态博客解决的是维护问题

静态博客构建后就是 HTML、CSS、图片和少量脚本。没有后台登录入口，也没有数据库连接，出问题的面少很多。

对个人内容站来说，这已经够用：

- 文章用 Markdown 写；
- GitHub 负责保存历史版本；
- Cloudflare Pages 负责构建、HTTPS 和 CDN；
- 域名解析也可以交给 Cloudflare。

真正需要你长期投入的，是内容本身。

## 什么时候不适合静态博客

如果你要做会员系统、评论系统、在线支付、复杂表单，静态博客就不是完整答案。

但这不代表一开始就要上 WordPress 或自建 VPS。更稳的做法是先把博客跑起来，等真实需求出现后，再接评论、邮件订阅或轻量后台。

## 我会怎么搭

这个博客使用 Astro。原因很简单：它适合内容站，默认输出静态文件，部署到 Cloudflare Pages 很直接。

后续写文章时，只要在 `src/pages/blog/` 里新增 Markdown 文件，首页、归档页和 RSS 都会自动更新。

先把发布链路跑通，再去优化主题、域名和搜索收录。
